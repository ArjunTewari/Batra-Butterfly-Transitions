Tables:

GET /retailers

Daily refresh (every 24 hrs):
recompute all outstanding balances

Frontend:
Retailer list with:
outstanding
credit limit
overdue badge
Retailer detail page:
ledger timeline
payment history
charts (optional)

For each retailer:

total_purchase_last_30_days

avg_order_value
order_frequency
last_order_gap
Derived Metrics:
growth_rate = recent avg vs past avg
potential_score = weighted score:
(frequency + consistency + payment behavior)

API:
GET /analytics/top-retailers

Frontend:
Dashboard cards:
Top 10 retailers
Under-performing retailers
Filters:
time range
category (future)

4. AI-ASSISTED PHOTO STOCK MANAGEMENT
Backend Logic:
Tables:
products (id, article_code, name, price, image_url)
stock_movements (id, product_id, type [in/out], quantity, image_url, date)
Flow:
User uploads image
Backend sends image to AI API
AI returns:
predicted_article_code
confidence_score
System matches with product DB
User confirms
Stock updated
API:
POST /stock/upload-image
POST /stock/confirm
GET /stock
Important Logic:
Do NOT auto-update stock without confirmation
Store image + mapped SKU for future reuse (no repeat AI cost)
Frontend:
Upload UI
Preview image
Show AI suggestion
Confirm / edit SKU + quantity
Submit
